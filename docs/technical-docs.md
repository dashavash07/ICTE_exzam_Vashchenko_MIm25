# Технічна документація проєкту: 3D-координатна система (A.1.4)

## 1. Загальна архітектура проєкту
Проєкт побудований за модульним принципом із розділенням логіки, представлення та описової частини. 

### Структура файлів:
- `index.html` — головна сторінка-лендінг (презентація проєкту).
- `demonstracion.html` — сторінка безпосередньо 3D-тренажера, що містить DOM-елементи панелі керування та контейнер `<a-scene>`.
- `script.js` — винесена математична та інтерактивна логіка проєкту.
- `docs/` — папка з проєктною документацією (посібник користувача та технічна документація).

## 2. Стек технологій
- **A-Frame (v1.4.2):** Web-фреймворк для побудови віртуальної реальності (VR) та 3D-сцен. Працює поверх бібліотеки Three.js.
- **Three.js:** Використовується під капотом A-Frame для складних математичних розрахунків (зокрема, обчислення кватерніонів для повороту стрілок векторів).
- **JavaScript (ES6):** Обробка подій, математичні обчислення та динамічне маніпулювання DOM-деревом.
- **HTML5 / CSS3:** Побудова адаптивного інтерфейсу за допомогою модулів Flexbox.

## 3. Детальний опис роботи 3D-сцени (`demonstracion.html`)
Сцена ініціалізується за допомогою тегу `<a-scene embedded>`, що дозволяє інтегрувати 3D-середовище у звичайний div-контейнер (`.scene-container`). 

**Основні компоненти сцени:**
- **Освітлення:** `ambient` (загальне) та `directional` (спрямоване) для надання об'єктам тіней та об'ємності.
- **Осі координат:** Статичні `<a-entity>`, що складаються з циліндрів (ліній), конусів (стрілок) та `<a-text>` для підписів осей (X, Y, Z).
- **Площина:** Напівпрозорий елемент `<a-plane>` з властивостями `material="opacity: 0.4; transparent: true"` та `side="double"`, що слугує візуальною опорою, але дозволяє бачити вектори з від'ємними координатами.
- **Контейнер векторів:** Порожній `<a-entity id="vectors-container">`, куди JavaScript динамічно ін'єктує створені вектори.

## 4. Програмна логіка (`script.js`)

### 4.1. Система обертання сцени (Custom Drag Controls)
Стандартні елементи керування камерою (`look-controls`, `wasd-controls`) вимкнено для забезпечення стабільного кута огляду. Натомість реалізовано обертання всієї системи координат (елемент `#space-rotator`) навколо осі Y за допомогою подій миші та сенсорного екрана:
- Змінна `isDragging` відстежує стан затискання лівої кнопки миші або дотику.
- При русі вираховується `deltaX`, що додається до `currentRotationY`, змінюючи атрибут `rotation` головного контейнера.

### 4.2. Математичні обчислення
Обробка даних починається з функції `getVector(prefix)`, яка зчитує значення з полів вводу (парсинг через `parseFloat`) та формує об'єкт із властивостями `x, y, z`.

Реалізовано три основні оператори:
1. **Додавання:** `c.x = a.x + b.x`, `c.y = a.y + b.y`, `c.z = a.z + b.z`.
2. **Віднімання:** `c.x = a.x - b.x`, `c.y = a.y - b.y`, `c.z = a.z - b.z`.
3. **Скалярний добуток:** Обчислюється за формулою `(a.x * b.x) + (a.y * b.y) + (a.z * b.z)`. Повертає скалярне (числове) значення.

### 4.3. Динамічна візуалізація (`drawVector`)
Функція `drawVector(v, color, label, isResult)` відповідає за створення 3D-об'єктів:
1. **Лінія:** Створюється `<a-entity>` з атрибутом `line` від початку координат до точки `(v.x, v.y, v.z)`.
2. **Стрілка (вістря):** Створюється `<a-cone>`. Для правильної орієнтації конуса в просторі використовується API Three.js:
   - Створюється нормалізований вектор напрямку: `new THREE.Vector3(v.x, v.y, v.z).normalize()`.
   - Обчислюється кватерніон повороту відносно базової осі Y (`up`).
   - Метод `quaternion.setFromUnitVectors()` застосовується до `object3D` конуса після події `loaded`.
3. **Підпис:** Додається `<a-text>`, зміщений відносно кінця вектора для уникнення накладання тексту на геометричні фігури. Має властивість `side="double"` для видимості під час обертання сцени.

## 5. UX та оптимізація
- **Спрощення вводу:** Поля вводу мають приховані стандартні браузерні стрілки, що імітує класичний математичний запис у зошиті.
- **Очищення пам'яті:** Функція `clearScene()` видаляє старі DOM-елементи з контейнера перед кожною новою операцією, запобігаючи витоку пам'яті та захаращенню сцени.
